## What it is

A small icon-first button. Renders a `<button>` with muted default colors, a brighter hover state, and an "active" variant that reads as *currently-toggled*.

## Why it exists

Toolbars kept re-implementing the same button shape: a square hit area, a 1.5x icon, muted-default/bright-hover/tinted-active states. Extracting it removed 30+ lines of duplicated markup across PhotoViewer, CompareView, and a couple of header rows.

The interface is deliberately tiny (`onClick`, `active`, `children`, passthrough props) so it can host any icon from `@phosphor-icons/react` or `lucide-react` without caring which.

## Code

```tsx
// src/examples/btn.tsx
interface BtnProps extends Omit<ComponentPropsWithoutRef<'button'>, 'onClick' | 'children'> {
  onClick?: MouseEventHandler<HTMLButtonElement>;
  active?: boolean;
  children: ReactNode;
}

export default function Btn({ onClick, active, children, ...props }: BtnProps) {
  return (
    <button onClick={onClick} style={{ /* muted / hover / active */ }} {...props}>
      {children}
    </button>
  );
}
```

See the full example file at `src/examples/btn.tsx` in this repo.

## How it's used

- **Cairn / PhotoViewer** — zoom, favorite, share, and delete toolbar buttons
- **Cairn / CompareView** — swap, zoom in/out, fit controls
- **Artifex / PhotoViewer** — same shape, slightly different active background

## Gotchas

- **Must live at module scope.** Defining `Btn` inside another component's render path gives every re-render a fresh component identity, which unmounts and remounts every instance on every parent render. eslint-plugin-react-hooks catches this with `react-hooks/static-components`. See `artifex/bug-component-in-render-001` for the original incident.
- **No forwardRef.** If you need focus management or to point at the DOM node from a parent (e.g. tooltip anchors), wrap it or switch to `React.forwardRef`.
- **Hover state uses inline style.** Works everywhere, but means you can't theme it via CSS variables. Fine for teaching; for production, prefer a class-based approach with your theme tokens.
